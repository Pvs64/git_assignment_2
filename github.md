Repositories – Store and manage project files along with their complete version history.

Version Control (Git Integration) – Tracks changes in code and allows reverting to previous versions.

Branches – Enable working on different features or fixes without affecting the main code.

Pull Requests – Allow users to propose, review, and merge code changes collaboratively.

Issues – Help track bugs, tasks, and feature requests within a project.

GitHub Actions – Automates workflows like testing, building, and deploying code.

Forking – Creates a personal copy of another repository for independent development.

GitHub Pages – Hosts static websites directly from a repository.
